# Externalize tunables + restructure config — Design

**Date:** 2026-06-07
**Status:** Approved (brainstorming) → implementation planning
**Scope:** Move runtime config into the package, split `app_config.yaml`, make `*_config.yaml`
the single tuning surface per device, restructure hand poses per-finger, and consolidate
motor-ID identity into code. No behavior change to the robots — only *where* values live.

---

## 1. Guiding principle

Three categories of "param", handled differently:

| Category | Examples | Home |
|---|---|---|
| **Operator knobs** | step sizes, speeds, margins, load thresholds, settle/dwell times, COM ports | **Config YAML** (`*_config.yaml`) — the tuning surface |
| **Measured values** | `middle_pos`, DOF `limits`, encoder ranges/homing offsets | **Calibration artifacts** (regenerated by calibration, never hand-tuned) |
| **Hardware facts + identity + safety canon** | `STS3215_RESOLUTION`, motor IDs (IL-3), nominal voltages 12 V/5 V (IL-1) | **Code**, one copy each, with a `# SSOT` pointer comment |

Rule: editing a config YAML can never break the math or violate an Iron Law. Identity/physics
stay in code as the single source.

**Why the calibration files stay separate (answers "why do we need them if we have `*_config.yaml`"):**
calibration files are measurement *outputs*, not tuning surfaces — you never hand-edit
`middle_pos`; you run `middle_calib`/`range_calib` (or lerobot's `calibrate`) and the hardware
measures them. After connection/speeds/`id` move out, **zero data is duplicated** between calib
and config. The arm JSON is *written and read by lerobot itself* — merging it would mean
reimplementing lerobot's calibration writer (a YAGNI/IL-2 violation). Symmetric model:
**each device = one `*_config.yaml` + one calibration artifact.**

## 3. Identity canon → code (the real DRY fix)

The motor name↔ID mapping was duplicated across code *and* data. Consolidate to one code copy
per device; calibration artifacts hold only measured values.

- **Arm:** the canonical motor table already lives in `SO101FollowerNoGripper` (`Motor(1,…)…`,
  exists before any calibration; lerobot needs it to open the bus). Delete `_ARM_MOTOR_IDS` from
  `arm_torque.py` — it reuses the device-layer construction (`build_follower`/shared helper).
  Keep `ARM_JOINTS` as the single names+order tuple (already imported widely).
- **Hand:** add a finger→servo-ID table to the **hand device layer** (IL-3 canon):
  `index→(1,2), middle→(3,4), ring→(5,6), thumb→(7,8)`. `pose_resolver` and
  `middle_pos_by_id()` get IDs from this table, `middle_pos` from the (trimmed) calib YAML.
  Drop `ServoCalibration.id`.

## 4. Schema changes (`src/arm101_hand/config/`)

| File | Change |
|---|---|
| `app_config.py` | **Deleted** (`AppConfig`, `load_app_config` gone). |
| `arm_poses.py` → `arm_config.py` | New `ArmConfig` = `connection` · `safety` · `tuning` · `poses`. `load_arm_config`/`save_arm_config` (whole-model load-modify-save preserves all sections). |
| `hand_poses.py` → `hand_config.py` | New `HandConfig` = `connection` · `safety` · `tuning` · `poses`. Poses validated as 4 named fingers × 2 ints. `load_hand_config`/`save_hand_config`. |
| `calibration.py` | Drop `com_port`/`baudrate`/`timeout`/`speed`/`speeds` and `ServoCalibration.id`; bump required `schema_version` to 3; keep `fingers` (middle_pos + limits), `middle_pos_by_id()` (now id from code table), `limits_by_finger()`. |
| `__init__.py` | Update exports + docstrings. |

`connection`/`safety`/`tuning` pydantic models carry **`Field(description=…)`** for every knob —
the single documentation home (IL-7); inline YAML comments are not relied upon (save regenerates).

### 4(a) One file per device, machine-managed
`jog.py`/`capture_pose.py` save poses via whole-model `safe_dump` → inline YAML comments are
stripped. Accepted: keys are self-descriptive; per-knob meaning documented in `data/README.md` +
pydantic `Field` descriptions. Load-modify-save preserves all sections' *values* across a pose save.

### 4(b) Protocol micro-delays
`time.sleep(0.0002/0.005/0.01)` bus-sync delays are protocol timing, not operator knobs →
collapse to one named constant in the hand device layer (with a pointer comment). Operator-
meaningful settle/dwell times go to YAML `tuning`.

## 5. Blast radius (~20 files)

- **Path constants:** `scripts/device_setup.py` (`_REPO_ROOT`, `APP_CONFIG_PATH`→removed,
  `ARM_CONFIG_PATH` new location), `arm_torque.py`, `scripts/demos/grab_sequence.py`,
  `scripts/calibration/amazing_hand/{set_pose,jog}.py`. New base = `src/arm101_hand/data/`.
- **Hand scripts now read connection from `hand_config`:** `motor_reset`, `middle_calib`,
  `range_calib`, `finger_test`, `full_hand_test`, `set_pose`, `jog` (7) — they currently get
  `com_port/baudrate/timeout` from the calib YAML.
- **Diagnostics:** `scan.py` (per-device thresholds; hand port from `hand_config`),
  `show_calib.py` (hand port from `hand_config`). Nominal voltages 12 V/5 V stay in code (IL-1).
- **Arm motion scripts:** `sweep.py`/`set_pose.py`/`jog.py`/`capture_pose.py` read knobs
  (`_LOAD_*`, `_SETTLE_S`, `--margin` default) from `arm_config.tuning`.
- **`pyproject.toml`:** include `src/arm101_hand/data/*.yaml` as package data.
- **Tests:** delete `test_app_config.py`; update `test_device_setup.py`,
  `test_arm_poses_schema.py`/`_save.py`, `test_hand_poses_schema.py`/`test_hand_config_save.py`,
  `test_calibration.py`, `test_hand_pose_resolver.py` for new shapes/paths; add tests for
  `connection`/`safety`/`tuning` + per-finger poses + finger→id table.
- **Docs:** `data/README.md` (rewrite), `CLAUDE.md` (paths, §3 tree, §4 workflows, §5 IL-5),
  and `docs/conventions/` references to `app_config.yaml`/`data/`.

## 6. Iron Law touchpoints

- **IL-2** — arm JSON stays lerobot-owned; no fighting the vendored library.
- **IL-3** — motor IDs become the single code canon (arm subclass + hand finger table).
- **IL-5** — calibration files stay at their mandated paths; config files move into the package
  but remain in-tree, version-controlled, written only by scripts. `CLAUDE.md` IL-5 line +
  `00-iron-laws.md` updated to point at `src/arm101_hand/data/` for config (calibration paths
  unchanged). Run `iron-law-audit` before the PR.
- **IL-6** — touches both devices → one atomic commit.
- **IL-7** — each fact (knob meaning, motor ID, schema) has exactly one home; docs are pointers.

---

## 7. Out of scope (YAGNI)

- No schema migration tooling — values are migrated by hand once in this change.
- No teleop/policy work.
- No change to lerobot's calibration format or the arm JSON contents.
